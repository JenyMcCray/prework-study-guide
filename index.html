HTML

  • !DOCTYPE html declaration tells the browser what document type to expect; declarations are not case-sensitive.
  • Element - refers to the entire element from opening tag to closing tag.
  • Tag - refers to what appears inside the angle brackets.
  • Attribute - defines additional information about an element; always defined at the beginning element, never the closing element.
  • Child elements - elements that are nested inside other elements; element holders are parent elements.
  • Semantic HTML elements - the name of the element describes its content and the role it plays on a webpage; makes HTML easier to read by other developers and the browser; also plays an important role in accessibility.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • Selector - defines the element or attributes to which the rules, or declarations, will apply.
  • Declaration - contains two important components: the CSS property we want to apply and the value of that property.
  • Margin - indicates how much space we want around the outside of an element; not visible on a page, just acts as a buffer.
  • Padding - adds space around the content inside an element; makes the entire element bigger.
  • * selector - applies a rule to all the elements visible on a page.

Git

  • git status - checks what branch we are currently on.
  • git checkout -b branch-name - creates a new branch and switches to it.
  • git add -A - stages changes; selects all files in the working branch to commit.
  • git commit -m "added code to HTML file" - commits the changes (-m adds a message)

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Comments - snippets of text that can be added along with the code; browser ignores text marked as comments
  • Operator - mathematical symbol that produces a result based on two vales (or variables)
  • Conditionals - code structures used to test if an expression returns true or not; a very common form of conditionals is the if…else statement
  • Arrays - a single variable that is used to hold a group of related data; regular objects for which there is a particular relationship between integer-keyed properties and the length property; perfect candidate to represent ordered lists; can store different types of data such as strings, numbers, etc.
  • Functions - ways of packaging functionality that you wish to reuse; possible to define a body of code as a function that executes when you call the function name in your code; good alternative to repeatedly writing the same code; functions look like variable names but are followed by parentheses ()
